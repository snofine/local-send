# 📡 local-send

Легковесная консольная утилита для **быстрого обмена файлами в локальной Wi-Fi сети**. Один Python-скрипт — без серверов, без регистрации, без облака. Запустил на компьютере, отсканировал QR-код телефоном — и обмениваешься файлами через браузер.

## ✨ Возможности

- 📲 **Мгновенный доступ** — при запуске определяет локальный IP и выводит в консоль **QR-код** в ANSI-графике. Сканируешь телефоном — открывается веб-интерфейс.
- 📤 **Отправка файлов на ПК** — drag-and-drop зона + выбор файлов, **мульти-загрузка** (несколько файлов сразу), индикатор прогресса в реальном времени.
- 📝 **Быстрые заметки** — текстовое поле для ссылок/заметок, которые сохраняются на ПК в текстовый файл.
- 📥 **Скачивание с ПК** — динамический список файлов из папки `uploads/`, скачивание в один клик.
- 🗄️ **Без лимита размера** — поблочная потоковая передача; гоняет большие видео и логи в несколько ГБ без переполнения памяти.
- 🔀 **Защита от перезаписи** — при совпадении имён добавляются суффиксы `_1`, `_2`, …
- 🌗 **Тёмная/светлая тема** — веб-интерфейс запоминает выбор в браузере.
- 🪶 **Один файл** — весь код (включая HTML/CSS/JS) упакован в `main.py`.

## 🚀 Установка

Нужен только **Python 3.10+**. Установить зависимости один раз:

```bash
pip install fastapi uvicorn qrcode python-multipart
```

Если запустишь `main.py` без установленных пакетов, скрипт сам подскажет нужную команду и завершится.

<details>
<summary>⚠️ Если PyPI заблокирован (корпоративный прокси)</summary>

Используй зеркало:

```bash
pip install fastapi uvicorn qrcode python-multipart -i https://pypi.tuna.tsinghua.edu.cn/simple/
```

</details>

```bash

```

В консоли появится баннер с IP, портом, URL и **QR-кодом**:

```

  Порт : 8080
  URL  : http://192.168.3.29:8080
════════════════════════════════════════════════

════════════════════════════════════════════════

  █ ███ █ ▄ ▄█ ▄ ▀█ █ ███ █
```

## 📂 Структура папок

При первом запуске рядом со скриптом создаются две директории:

| Папка          | Назначение                                                              |
| -------------- | ----------------------------------------------------------------------- |
| `downloads/`   | Сюда **сохраняются** файлы и заметки, отправленные с телефона на ПК.    |
| `uploads/`     | Положи сюда файлы на ПК — они появятся в списке для скачивания на телефон. |

```
local-send/
├── main.py          # вся утилита (бэкенд + веб-интерфейс)
├── downloads/       # принятое с телефонов
│   ├── photo.jpg
│   └── note_20260624_175227.txt
└── uploads/         # расшарено с ПК
    └── big_video.mp4
```

## 🌐 Веб-интерфейс

Страница построена на чистом **HTML5 + Tailwind CSS (CDN) + Vanilla JS** — без сборки, без фреймворков.

### Блок 1 — Отправить на ПК
- 🟦 Зона drag-and-drop (или тап для выбора файлов на мобильном).
- 📊 Progress bar на основе `XMLHttpRequest.upload.onprogress` — реальный % загрузки.
- 📝 Поле для заметок/ссылок → сохраняется в `downloads/note_<timestamp>.txt`.

### Блок 2 — Скачать с ПК
- 📋 Автоматический список файлов из `uploads/` с размером и датой изменения.
- ⬇️ Кнопка «Скачать» — скачивание в один клик.
- ↻ Кнопка «Обновить» — перечитать список.

## 🔌 HTTP API

Все эндпоинты принимают/отдают JSON, кроме скачивания файла.

| Метод   | Путь                  | Описание                                              |
| ------- | --------------------- | ----------------------------------------------------- |
| `GET`   | `/`                   | Веб-интерфейс (HTML).                                 |
| `GET`   | `/health`             | Проверка живости → `{"ok": true}`.                    |
| `GET`   | `/files`              | Список файлов из `uploads/`.                          |
| `GET`   | `/download/{name}`    | Скачать файл (потоковая отдача).                      |
| `POST`  | `/upload`             | Принять файлы (`multipart/form-data`, поле `files`).  |
| `POST`  | `/notes`              | Сохранить текстовую заметку (поле `text`).            |

### Примеры

```bash
# Проверка
curl http://192.168.3.29:8080/health

# Загрузить файл
curl -X POST http://192.168.3.29:8080/upload \
     -F "files=@photo.jpg" \
     -F "files=@notes.txt"

# Отправить заметку
curl -X POST http://192.168.3.29:8080/notes \
     --data-urlencode "text=Привет с телефона!"

# Скачать файл
curl -O http://192.168.3.29:8080/download/big_video.mp4
```

## ⚙️ Конфигурация

Все настройки — константы в начале `main.py`:

```python
DOWNLOADS_DIR = BASE_DIR / "downloads"
UPLOADS_DIR   = BASE_DIR / "uploads"
CHUNK_SIZE    = 1024 * 1024   # 1 MiB — размер блока при чтении/записи
PREFERRED_PORT = 8080         # предпочитаемый порт
```

## 🔒 Безопасность

- Утилита работает **только в доверенной локальной сети** — без аутентификации и TLS.
- Защита от path-traversal: имена файлов очищаются до `basename`.
- Не открывай порт наружу (в интернет) без дополнительной защиты.

## ❓ Решение проблем

<details>
<summary><b>Телефон не открывает страницу по QR</b></summary>

- Убедись, что ПК и телефон в **одной Wi-Fi сети**.
- Проверь, что брандмауэр Windows/Линукс не блокирует порт 8080.
- Попробуй открыть URL вручную в браузере телефона: `http://<IP>:8080`.

</details>

<details>
<summary><b>QR-код отображается криво</b></summary>

- Используй терминал с поддержкой Unicode и достаточным размером окна (не менее 40×20 символов).
- На Windows рекомендуются Windows Terminal или PowerShell 7.

</details>

<details>
<summary><b>Не определяется правильный IP</b></summary>

- Если компьютер подключён и к Wi-Fi, и к проводной сети, выбранный IP может отличаться. Укажи нужный вручную в `get_local_ip()`.

</details>
